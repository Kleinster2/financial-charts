# Ambient AI

Computing that disappears into the environment — proactive, context-aware, requiring no explicit interaction. The paradigm shift from "using computers" to "environments that assist you."

**For wearables, glasses, rings, and screenless devices → see [[Screenless AI]]**

## What is Ambient AI?

| Aspect | Description |
|--------|-------------|
| **Core idea** | AI embedded invisibly in environments, not devices you hold |
| **Interaction** | Proactive, anticipatory — no prompts needed |
| **Inputs** | Sensors, voice, vision, motion, biometrics, context |
| **Output** | Actions happen automatically; environment adapts |

**The shift:** From app-centric (open app → give command → get result) to environment-centric (space senses context → acts on your behalf).

## Infrastructure

Ambient AI requires:

| Component | Role |
|-----------|------|
| **Edge compute** | Low-latency local processing |
| **Cloud** | Model hosting, cross-device learning |
| **Sensors** | Environmental awareness (temp, motion, sound, vision) |
| **Connectivity** | WiFi 6/7, Matter protocol, 5G |
| **AI models** | Multimodal, context-aware, predictive |

**The invisible cloud:** While ambient computing feels local, it's powered by massive cloud infrastructure (AWS, Azure, GCP) that processes data streams from billions of devices.

## Challenges

| Challenge | Description |
|-----------|-------------|
| **Privacy** | Always-on sensors, data collection |
| **Interoperability** | Alexa vs Google vs Apple silos |
| **Security** | Attack surface across distributed systems |
| **Trust** | Users must trust AI acts correctly (no screen to verify) |
| **Cost** | Full ambient systems expensive to deploy |
