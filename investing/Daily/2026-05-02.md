### Infrastructure: Tailscale-accessible Quartz viewer + auto-restart watchdog

Stood up phone access to the investing vault over Tailscale. Updated the Quartz `Plugin.LightweightCharts` `apiBaseUrl` from `localhost:5000` to `gilssurface:5000` so iframe charts resolve from any tailnet device. Wrote `serve_static.py` (a `SimpleHTTPRequestHandler` subclass that maps `/Foo` → `Foo.html`) on top of the Quartz `public/` snapshot, since Quartz emits flat HTML files but renders extensionless `<a href>`s — plain `python -m http.server` 404s on internal links. Quartz rebuilt twice (~30 min each) to bake the new `apiBaseUrl` into all pages.

Registered a Task Scheduler watchdog (`FinancialCharts-OnLogon` + `FinancialCharts-Watchdog` every 5 min) so Flask :5000 and the static server :8080 self-heal on crash and restart at logon. Verified by killing the static server and watching the watchdog re-spawn it. Required `-RedirectStandardOutput`/`-RedirectStandardError` on `Start-Process` because `pythonw.exe` blocks on `print()` with no attached console — the symptom was a process bound to the port that never served a request. Documented in `docs/ops/server-watchdog.md` and committed `bd065fc`.

Local Quartz patches snapshot-committed to the `quartz-investing` repo (`b065dbf`, NOT pushed — origin is upstream Quartz) so a stray reset doesn't lose the lwcharts transformer, frontmatter try/catch, Windows alias-char sanitization, and config edits.
