## Framework / tooling

### Cluster-validation falsification gap closure

Closed the four statistical gaps identified yesterday in the cluster validation framework. Added four new scripts plus doc + checklist updates:

- `scripts/cluster_permutation_test.py` — permutation p-values for intra-corr + PC1 against two nulls: independence (shuffle within ticker) and random-basket (sample N tickers from the broader DB universe). The random-basket null is the meaningful test.
- `scripts/cluster_holdout_test.py` — temporal train/test split with regime-durability verdict (STABLE / STRENGTHENING / WEAKENED / REGIME-DEPENDENT).
- `scripts/cluster_threshold_scan.py` — sweep distance thresholds 0.20-0.70 and report the stable threshold range (verdict ROBUST / MODERATELY ROBUST / FRAGILE / BOUNDARY-DEPENDENT).
- `scripts/cluster_registry.py` — CSV log + Bonferroni / Benjamini-Hochberg correction across all registered cohorts. Other scripts auto-append.

Smoke-tested on the two canonical examples (`rklb.yaml` validated, `mag7.yaml` falsified). The new diagnostics produced sharp results:

| Diagnostic | Space pure-plays | Mag 7 |
|---|---|---|
| Random-basket p (intra) | < 0.001 | 0.010 |
| Random-basket p (PC1) | < 0.001 | 0.027 |
| Holdout test/train ratio | 1.37 STRENGTHENING | 0.44 REGIME-DEPENDENT |
| PC1 loadings stability | 0.56 | -0.08 (factor flipped) |
| Threshold stable width | 0.05 (FRAGILE, [0.45, 0.50]) | 0.00 (BOUNDARY-DEPENDENT) |

The holdout test surfaced a finding worth flagging: Mag 7 was a real cluster in 2024 (intra-corr 0.72) but the factor structure flipped by 2025-2026 (intra-corr 0.32, PC1 loadings corr -0.08). The "Mag 7 as basket" claim is now empirically false in current trading regime, not just below the conventional thresholds.
