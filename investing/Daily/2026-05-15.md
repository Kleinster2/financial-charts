### Wide-table silent-failure hardening (surgical fix)

Followed up with the cheap interim fix flagged above: `update_market_data.py` and `download_all_assets.py:update_sp500_data()` now propagate wide-table-write failure as a non-zero exit code, with two `[STALE-WIDE]` lines written to stderr. The downstream pipeline (DuckDB shadow, metadata refresh) still completes — only the orchestrator exit code is affected. This means `scripts/morning_scan.py` (which already inspects the exit code of the subprocess call to `update_market_data.py --lookback 5`) will now print a `WARN: update_market_data exited 1` line on the next wide-table failure instead of silently succeeding. Verified via a synthetic test at `scripts/_test_wide_fail.py` that monkey-patches `pd.DataFrame.to_sql` on staging tables to raise — confirmed exit code 1 + `[STALE-WIDE]` markers on stderr. Full wide-table deprecation (86 reader files to migrate) is deferred as a larger project.
