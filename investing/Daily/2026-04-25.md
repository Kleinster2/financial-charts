### Quartz viewer ship — wrap-up (Apr 25, 2026)

Continued the Apr 24 Quartz-as-viewer migration. Today was polish + infrastructure hardening, not vault-content research:

- Iterated chart styling for live iframe widgets via `window.INTERACTIVE` in `chart_render.html`: `autoSize: true`, axis fontSize 12 (was 20 for screenshots), compact legend at top-left with a white-pill background, latest-data date carried as a legend item, no separate bottom-left date badge. The PNG screenshot path stays bit-for-bit identical when `format` is omitted.
- Added [[index]] (`investing/index.md`) — thin alias to [[Home]] so `localhost:8080/` resolves instead of 404. Quartz expects `content/index.md`; the vault uses [[Home]] as the actual hub. [[Home]] remains canonical; [[index]] just resolves the bare URL.
- Decommissioned the `obsidian-chart-refresh/` plugin: deleted source (10 files, ~1,400 LOC). Its only role was firing on Obsidian's `workspace.on("file-open")` event to refresh chart PNG embeds. Now that Quartz iframes hit `/api/chart/lw?format=html` on every page load, freshness equals or exceeds what the plugin provided. Filename-parsing logic was already ported into the Quartz transformer (`lwcharts.ts`).
- Documented the Quartz install in `docs/quartz-viewer.md`: layout, run command, four local Quartz patches (lwcharts.ts, frontmatter.ts try/catch, helpers.ts Windows-char sanitization, quartz.config.ts plugin registration + ignorePatterns), chart-registry format, known issues, when-to-rebuild list, curl-based health check.

Stale-but-accurate references to the deleted plugin remain in `docs/chart-api.md` and `docs/obsidian-chart-refresh-plugin.md`. The parsing rules they describe carry over to Quartz's `lwcharts.ts` — only the actor changed. Flagged as a follow-up for a later docs refresh pass.
