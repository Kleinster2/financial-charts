### Frontmatter wikilink cleanup — 166 files (Apr 25, 2026)

Closed the highest-impact remaining Quartz-viewer follow-up: 45 vault notes had wikilinks inside YAML `aliases:` arrays (e.g. `aliases: [[[Burlington]] Northern Santa Fe]`) that strict YAML couldn't parse, plus another ~120 files had wikilinks inside other quoted frontmatter fields (e.g. `parent_concept: "[[Generative AI]]"`).

The Quartz `frontmatter.ts` plugin had a try/catch I added earlier that tolerated the parse errors but fell back to *empty frontmatter* — meaning those notes lost their aliases, tags, and proper title in the Quartz view. UX cost was real: typing an alternate name like "BNSF" wouldn't resolve to [[BNSF]] for users on the Quartz site.

Fix: `scripts/fix_alias_wikilinks.py` — small Python utility that strips `[[` and `]]` characters from anywhere inside the frontmatter block (between `---` markers). Body wikilinks untouched. Result: clean YAML strings, no functional regression (frontmatter values were never rendered as wikilinks anyway in either Obsidian or Quartz; the brackets were noise).

166 files modified across `investing/`. Spot-checked [[BNSF]] (aliases now `[Burlington Northern Santa Fe]`), [[01.AI]] (parent_concept now `"Generative AI"`), [[Andreessen Horowitz]] (aliases `[a16z, Andreessen]`), etc. No script in the repo reads `parent_concept` / `parent_actor` frontmatter fields, so no downstream consumer depends on the bracket syntax.

The Quartz `frontmatter.ts` try/catch patch stays in place as a defensive net for future edits, but should now hit the catch path zero times in normal operation. Build warnings should drop accordingly.

### Quartz duplicate-tag-row fix (Apr 25, 2026)

Vault notes use inline `#sector #drones ...` hashtag lines at the top of bodies (Obsidian convention). Quartz's ObsidianFlavoredMarkdown extracts those into the frontmatter tag set AND renders them inline as tag chips in the body — and `Component.TagList()` then displays the same set above the title. Result: two identical rows of tag chips above each note. Visible on Drones, Defense Primes, every note with inline hashtags.

Fix: removed `Component.TagList()` from `defaultContentPageLayout.beforeBody` in `quartz.layout.ts`. Inline tags in the body still render as navigable chips (clicking goes to `/tags/<name>`); the duplicate header row is gone. This is a fifth local Quartz patch — documented in `docs/quartz-viewer.md` under "Local Quartz patches" alongside the existing four (lwcharts.ts, frontmatter.ts try/catch, helpers.ts Windows-char sanitization, quartz.config.ts).

Layout-config changes trigger a full Quartz rebuild from scratch; the running `npx quartz build --serve` re-emits all 5,984 files (~30 min cold). After rebuild, only one row of tags shows.
