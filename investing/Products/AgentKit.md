AgentKit is [[Coinbase]]'s open-source SDK for building AI agents with native cryptocurrency wallet capabilities, forming the infrastructure layer for the emerging "agentic commerce" thesis where autonomous AI agents transact on behalf of users.

## Synopsis

AgentKit provides developers with a framework to create AI agents that can hold crypto wallets, execute on-chain transactions, and interact with DeFi protocols autonomously. The SDK integrates with [[Coinbase]]'s MPC (multi-party computation) wallet infrastructure, meaning agents can control wallets without any single party holding full private keys — a security architecture designed to make autonomous agent transactions viable at scale. The product launched in late 2024 and underpins Coinbase's broader "Agentic Wallets" initiative and the x402 HTTP payments protocol, which enables AI agents to pay for API calls, data access, and services using crypto micropayments.

The strategic bet is that the AI agent economy will need native payment rails. If AI agents become common intermediaries — booking travel, purchasing services, executing trades — they need a way to hold and transfer value that does not require human approval for every transaction. Traditional payment rails (credit cards, bank transfers) are poorly suited for autonomous, high-frequency, small-value transactions between machines. Crypto rails (instant settlement, programmable, no intermediary approval) fit the use case better, and AgentKit is Coinbase's play to own that infrastructure layer. The competitive landscape includes [[Stripe]]'s agent payment APIs, [[PayPal]]'s stablecoin efforts, and various DeFi protocols building agent-native tools. For Coinbase, AgentKit is less about immediate revenue and more about positioning: if even 5% of AI agent transactions flow through crypto rails, the total addressable market could be enormous. [[Alesia Haas]] (Coinbase CFO) has framed it as a multi-year bet on AI-crypto convergence.
